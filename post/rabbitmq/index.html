RabbitMQ

集成RabbitMQ，实现系统间的数据交换

认识RabbitMQ

概念

MQ(Message Queue)。

RabbitMQ是AMQP(高级消息队列协议)的实现。

使用Erlang语言编写。

RabbitMQ的处理流程：

为什么要用

  • 系统解耦
  • 异步调用
  • 流量削峰

使用场景

  1. 推送通知

    “发布/订阅"功能来实现通知功能。

    “发布/订阅"功能支持三种模块：一对一、一对多、广播。

  2. 异步任务

    如果某个小任务很费时，且延迟执行并不影响整个任务，则可以将该任务放入消息队列中去处理，以便加快请求响应的时间。

  3. 多平台应用的通信

    消息队列是无关平台和语言的，RabbitMQ因此可以适合作为多个应用之间的松耦合的接口，且不需要发送方和接收方同时在线。

    这样最大限度地减少程序间的相互依赖，提高系统的可用性及拓展性。

    RabbitMQ提供两种事务模式：

    • AMQP事务模式；
    • Confirm事务模式；

  4. 消息延迟

    利用RabbitMQ消息队列延迟功能，可实现订单、支付过期定时取消功能。

    延迟队列存储延迟消息，当消息被发送以后，消费者不是立即拿到消息，而是等待指定时间后才拿到这个消息进行消费。

    当然，死信、计时器、定时任务也可以实现延迟或定时功能，但是需要开发者去处理。

  5. 远程过程调用

    在实际的应用场景中，有时需要一些同步处理，以等待服务器端将消息处理完成后再进行下一步处理，这相当于RPC(远程过程调用)。RabbitMQ也支持RPC。

特性

  • 信息确认
    • 自动应答：接收端从队列接收信息时，会自动发送应答消息给服务器端
    • 手动应答：需要开发人员手动调用方法告诉服务器端已经收到
  • 队列持久化：是否持久化，要看持久化设置；
  • 信息持久化：设置properties.DeliveryMode值（1：非持久，2：持久）；
  • 消息拒收：接收端可以拒收消息，而且在发送”reject"命令时，可以选择是否要把拒收的消息重新放回队列中；
  • 消息的Qos：在接收端设置的。加上如"channel.BasicQos(0,1,false);“的代码即可；

RabbitMQ基础概念

生产者、消费者和代理

  • 生产者：消息创建者，负责创建和推动数据到消息服务器
  • 消费者：消息接收方，用于处理数据和确认消息
  • 代理：RabbitMQ本身，扮演”快递“的角色，本身不生产消息

RabbitMQ只是为生产者和消费者提供发送和接收消息的API

消息队列

多个消费者可以订阅同一个Queue

  • 持久性
  • 自动删除
  • 惰性
  • 排他性

RabbitMQ的6种工作模式

简单模式

只有一个消费者、一个生产者、一个队列。

只需要配置主机参数，其他参数使用默认值即可通信。

工作队列模式

多个消费者，为了保证消费者之间的负载均衡和同步，需要在消息队列之间加上同步功能。

交换机模式

简单模式和工作队列模式也使用了交换机，只是采用了默认设置的交换机。交换机的参数是可以设置的。

如果消息配置的交换机参数和RabbitMQ队列绑定的交换机名称相同，则转发，否则丢弃。

Routing转发模式

交换机要配置为direct类型，转发的规则变为检查队列中的routing key值。如果roting key值相同，则转发，否则丢弃。

主题转发模式

交换机配置为topic类型，routing key配置失效。发送到主题交换机的信息，不能是任意routing key ，它必须是一个单词的列表，用逗号分隔。特点是可以模糊匹配，匹配规则为 *（星号）可以代替一个词；＃（＃号）可以代替零个或更多的单词。

RPC模式

远程调用的场景下。如果一个应用程序需要另外一个应用程序来最终返回运行结果，那这个过程可能是比较耗时的操作，使用 RPC 模式是最合适的。

工作总结

工作模式 特点
简单模式 只有一个生产者，一个消费者
工作队列模式 一个生产者，多个消费者，每个消费者获取到的消息唯一
订阅模式 一个生产者发送的消息会被多个消费者获取
路由模式 发送消息到交换机，并且要指定路由key，消费者再将队列绑定到交换机需要指定路由key
topic模式 根据主题进行匹配此时，此时队列需要绑定在一个模式上，”#”匹配一个词或多个词，“*”至匹配一个词

AmqpTemplate接口

Spring AMQP提供了操作AMQP协议的模板类AmqpTemplate，用于发送和接收消息

发送消息

  1. send方法
  2. convertAndSend方法：简单了的send方法，可以自动处理消息的序列化

接收消息

两种方式：

  • 直接去查询获取消息，调用receive方法；
  • 异步接收，通过注册一个Listener监听器来实现消息接收。接收消息需要指定队列，或设置默认的队列；

直接接收POJO（代替消息对象）的方法receiveAndConvert并提供了各种的MessageConverter用来处理返回的Object对象。

Sping-Rabbit 版本开始，Amqptemplate也提供了receiveAndReply方法来异步接收、处理及回复消息。

package workqueues;

import com.rabbitmq.client.Channel;
import com.rabbitmq.client.Connection;
import com.rabbitmq.client.ConnectionFactory;

/**
 * Created by Tang on 2021/5/28
 */
public class NewTask {

    private final static String TASK_QUEUE_NAME = "task_queue";

    public static void main(String[] args) throws Exception {
        //创建连接工厂
        ConnectionFactory factory = new ConnectionFactory();
        //设置RabbitMQ地址
        factory.setHost("139.196.19.64");
        factory.setUsername("admin");
        factory.setPassword("password");
        //建立连接
        Connection connection = factory.newConnection();
        //获得信道
        Channel channel = connection.createChannel();
        //声明队列
        channel.queueDeclare(TASK_QUEUE_NAME, true, false, false, null);
        for (int i = 0; i < 10; i++) {
            /**
             * 平均发送消息
             */
            /*String message;
            message = i + "...";
            channel.basicPublish("", TASK_QUEUE_NAME, null, message.getBytes("UTF-8"));
            System.out.println("发送了消息:" + message);*/

            /**
             * 不平均发送消息
             */
            String message;
            if (i % 2 == 0){
                message = i + "...";
            } else{
                message = String.valueOf(i);
            }
            channel.basicPublish("", TASK_QUEUE_NAME, null, message.getBytes("UTF-8"));
            System.out.println("发送了消息:" + message);
        }
        //关闭连接
        channel.close();
        connection.close();
    }
}

消费者

开启多个进行消费

package workqueues;

import com.rabbitmq.client.*;

import java.io.IOException;

/**
 * Created by Tang on 2021/5/28
 */
public class Worker {

    private final static String TASK_QUEUE_NAME = "task_queue";

    public static void main(String[] args) throws Exception {
        //创建连接工厂
        ConnectionFactory factory = new ConnectionFactory();
        //设置RabbitMQ地址
        factory.setHost("139.196.19.64");
        factory.setUsername("admin");
        factory.setPassword("password");
        //建立连接
        Connection connection = factory.newConnection();
        //获得信道
        final Channel channel = connection.createChannel();
        //声明队列
        channel.queueDeclare(TASK_QUEUE_NAME, true, false, false, null);
        //消费消息
        System.out.println("开始接收消息");

        //希望处理的数量为1,处理完之后才能执行下一个
        channel.basicQos(1);

        channel.basicConsume(TASK_QUEUE_NAME, false, new DefaultConsumer(channel) {
            //收到消息后会执行的函数
            @Override
            public void handleDelivery(String consumerTag, Envelope envelope, AMQP.BasicProperties properties, byte[] body) throws IOException {
                String message = new String(body, "UTF-8");
                System.out.println("收到消息: " + message);

                try {
                    doWork(message);
                } finally {
                    System.out.println("消息处理完成");
                    //手动确认消息
                    //channel.basicConsume 的true自动确认需要改为false
                    channel.basicAck(envelope.getDeliveryTag(), false);
                }

            }
        });
    }

    private static void doWork(String task) {
        char[] chars = task.toCharArray();
        for (char ch : chars) {
            if(ch == '.'){
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }

}

交换机类型

Exchange(交换机)包含4种类型：direct、topic、fanout、headers。不同类型代表绑定到队列的行为不同

四种类型模式

  • fanout：广播，这种模式只需要将队列绑定到交换机上即可，是不需要设置路由键的
  • direct：直接，根据 RoutingKey（路由键） 匹配消息路由到指定的队列
  • topic：生产者指定 RoutingKey （路由键）消息根据消费端指定的队列通过模糊匹配的方式进行相应转发
  • headers：根据发送消息内容中的 headers 属性来匹配（几乎不用）

fanout模式

路径：fanout

direct模式

路径：direct

消费者根据日志的不同等级去接收

topic模式

根据复杂条件匹配消息路由到指定的队列

路径：topic

规则：

  • *可以代替一个单词
  • ＃可以替代零个或多个单词
